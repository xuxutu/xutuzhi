本地仓库



1.首先 git config --list：在终端输入这个指令可以看到自己的用户名和邮箱。
[user]

        name = xuxutu
        email = 1355124336@qq.com

[core]
    editor=vim

2.如果你的Git还没有初始化，可以通过这个指令：
$ git config --global user.name “输入你的用户名”
$ git config --global user.email “输入你的邮箱”

3.如果已经初始化了而需要修改
  git config --global --replace-all user.email “我的邮箱”
  git config --global --replace-all user.name “用户名”
  最后再用git config --list

4.初始化目录,将目录变为git管理的仓库
  git init

5.将文件添加到暂存区
  git add . 

6.将文件从暂存区提交到本地仓库
  git commit -m "write a note file"
  git commit --amend 修改提交信息 
7.查看改动
  git diff 查看本地仓库的改动
  git show 查看上次commit后的改动
  git log -p 查看历次cmmit后的改动
  git log --pretty=oneline(按行显示提交说明)

8.退回到上一个版本
  git reset --hard HEAD^
  git reset --hard 79d43（返回指定版本）

9.查看工作区和版本库里的区别
  git diff HEAD -- gitnotes.txt

10.撤销工作区的修改
  git checkout -- gitnotes.txt

11.撤销暂存区的修改
  git reset HEAD gitnotes.txt
  git checkout -- gitnotes.txt

12.删除文件
   12.1 从版本库中删除文件
        git rm test.txt    
  
13. 从版本库里恢复文件到工作区
   git checkout -- test

远程仓库管理

1.注册
  1.username=xuxutu
  2.useremail=1355124336@qq.com
  3.password=

2.将远程仓库映射为别名 origin
git remote add origin git@github.com:xuxutu/tree.git

3.将本地修改推送到远程
git push -u origin master

4.解决冲突问题
git pull origin master --allow-unrelated-histories

5.合并分支到当前分支
git merge dev 

6.删除分支
git branch -D dev 


1.git 原理
1.1 git add
   .git/objects 的内容 是暂存区的文件内容 
   1.1.1 git cat-file -t 类型 
   1.1.2 git cat-file -p 内容
   1.1.3 将文件内容通过哈希算法处理成一份字符串（键值对）

1.2 git commit
   .git/objects 多出两个文件，一个文件记录文件信息，一个文件记录作者，提交说明等

指针：commit-->tree-->blob
 
   1.改动工作目录的文件时，只有工作区发生变化
   2.git add . 在本地库生成blob文件，更新索引的指向
   3.git commit 新建tree，新建commit，master指向更新。

2.恢复干净的工作区
  git stash push -u（git stash top 后悔药）
